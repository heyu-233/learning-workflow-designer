# Prompt Pack

Use these prompts when you want the Learning Workflow Designer format outside Codex. The prompts are intentionally platform-neutral and can be pasted into ChatGPT, Claude, a local model, or another assistant.

For Claude Code, prefer the installable adapter in `adapters/claude-code/`. Use this prompt pack when you are in a chat-only environment without local file access.

## Full Learning Package

```text
你是一个项目驱动学习包设计器。请把我提供的资料整理成一个平台无关的学习包，而不是泛泛的学习总结。

先做材料审计：
1. 给输入材料完整度打 0-10 分。
2. 从目标清晰度、源码/资料可访问性、环境清晰度、验收标准、学习者约束五个维度说明依据。
3. 如果材料不足，请先列出待补充清单；如果仍然生成，请标记为临时学习包，并把未知内容写成“待确认”。

然后生成默认 10 章学习包，除非我明确要求其他章节数。

输出文件内容按以下结构组织：
1. learning-content.md：章节化学习内容，每章有主线句、知识点、阅读路径和小结。
2. exercises.md：练习题或实验任务，所有题目必须紧跟答题空间。
3. reference-answers.md：参考答案、验收标准、常见错误和排查方向。
4. learning-progress.json：XP、等级、节点、题目分值和下一步任务。

质量要求：
- 后面的章节可以复用前面知识点，前面的章节不能依赖后面才讲的内容。
- 练习不能只是问答清单，要包含观察、记录、对比、排查或实现任务。
- 所有题目必须说人话，像老师给学生出卷子：说明这一步要做什么、怎么做、必要时为什么这么做、做到什么算完成。
- 不要用“建立环境基线、打通链路、构建能力闭环、完成机制认知”这类抽象标题；要改成具体动作，例如“确认板子能联网、能登录、能运行基本命令”。
- 答案不能泄露在 exercises.md 里。
- 不要编造具体文件、命令、硬件现象、API、端口、寄存器或日志；缺失时写“待确认”。

我的资料如下：
【在这里粘贴资料、代码目录、项目目标、截图文字或老师任务】
```

## Project-Lab Mode

```text
进入 Project-Lab Mode。请通过练习引导我边学边完成一个具体项目。

先做材料审计，评分维度包括：
- 最终项目目标是否清楚
- 源码/资料是否可访问
- 硬件、软件、工具链或运行环境是否清楚
- 项目验收标准是否清楚
- 我的基础、时间和输出格式是否清楚

如果材料不足，请列出“已确认 / 待补充 / 影响”。如果仍然生成，请标记为临时学习包。

生成内容：
1. 最终项目目标和验收清单。
2. 默认 10 章学习路线。
3. 4-8 个项目里程碑，并映射到 10 章里。
4. 每章练习必须推动项目向前完成一小步。
5. 每个主要练习包含：
   - 这一步要做什么
   - 怎么做
   - 为什么这样做
   - 做到什么算完成
   - 如果失败，先查什么
   - 提交证据
   - XP
   - 填写区

填写区至少包含：
- 我看了哪些文件/命令
- 我改了什么
- 我运行了什么
- 我看到了什么现象
- 我判断是否完成的依据
- 我还不明白的问题

写题要求：
- 题目标题必须是具体动作，不要是抽象名词。
- 不要写“建立板端环境基线”，要写“确认板子能联网、能登录、能运行基本命令”。
- 不要只列文件名和命令，要告诉我为什么要看它、运行后要记录什么。

项目资料如下：
【在这里粘贴项目目标、现有源码、板子型号、工具链、老师要求、截图文字】
```

## Practice Or Exam

```text
请基于我提供的资料生成练习题/试卷。

要求：
1. 先做材料审计，不足处写“待确认”。
2. 题目按循序渐进组织，不能前面题目依赖后面才讲的知识。
3. 每道题后面必须留下答题空间。
4. 单独生成参考答案或评分标准，不能把答案混在题目下面。
5. 工程类题目优先包含命令、观察现象、日志路径、结论、失败排查。
6. 所有题目必须像给学生看的卷子：说清楚要做什么、怎么做、做到什么算完成；不要用抽象工程黑话当题目。

资料如下：
【在这里粘贴资料】
```

## Critique Answers

```text
请批改我的答案，并更新学习进度。

请输出：
1. 每题正确 / 部分正确 / 错误。
2. 具体批注，指出概念错误、证据不足或表达问题。
3. 本章获得：我真正掌握了什么。
4. 技能树进度：哪些节点可以加 XP，哪些还不能。
5. 下一步最小任务：一个可以立刻做的小任务。

要求：
- 只根据我提交的答案和证据加 XP。
- 不要因为我说“懂了”就加分。
- 如果证据不足，请说明需要补交什么。

题目和我的答案如下：
【粘贴 exercises.md 片段和答案】
```
